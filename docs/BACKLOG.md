# Backlog развития TJR

## P0 (ближайшие)
- [x] Авто-отсечение шумовых сообщений (рекомендации кандидатов, курсы/вебинары) без ручных стоп-фраз.
- [ ] Явная пометка причины исключения в UI отчета (`исключено системой` / `исключено вашими фразами`).
- [ ] Фильтр в результатах: `все` / `только вакансии` / `пограничные`.
- [ ] UX прогресса сканирования: сделать более точный ETA и понятный индикатор активности для длинных каналов.
- [ ] История отчетов: хранить и открывать несколько последних отчетов (не только последний).

## P1
- [ ] Кнопки обратной связи в отчете: `Это шум` / `Это релевантно`.
- [ ] Локальный словарь антишум-фраз с редактированием в настройках.
- [ ] Авто-понижение score для сообщений без вакансийных сигналов (`ищем`, `вакансия`, `обязанности`, `условия`).
- [ ] Синонимы и роль-нормализация для топ-позиций (`CEO`/`гендиректор`/`управляющий директор`) с ручной настройкой.
- [ ] Пакетные действия в отчете: мультивыбор и массовый бан/разбан.

## P2
- [ ] Пресеты профилей поиска (несколько ролей под одной учеткой).
- [ ] Дедупликация одинаковых вакансий между каналами.
- [ ] Метрики качества выдачи (precision proxy по фидбеку).
- [ ] Еженедельный отчет по качеству подбора: доля шума, доля релевантных, каналы-лидеры по качеству.
- [ ] Умные уведомления: только high-confidence вакансии и отдельный канал для пограничных.
